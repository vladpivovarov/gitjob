Нас часто спрашивают

  • В чём ключевое отличие GitJob от аналогов?
    GitJob умеет сканировать репозитории вашего проекта, а затем репозитории разработчиков на Github и Stack Overflow. Так система понимает: 1) кандидаты с какими навыками нужны для вашего проекта; 2) на основе анализа профилей предлагает только тех кандидатов, чьи навыки подходят вашему проекту (стек навыков разработчика определяется не по заявленным в профиле, а по репозиториям и используемым библиотекам). Так вы получаете список кандидатов только с релевантным опытом.
  • Какие документы нужны для начала работы с GitJob?
    Мы используем полностью элекронный документооборот, для начала работы нам нужен стандартный набор документов – ИНН, ОГРН, подпись уполномоченного лица компании.
  • За что нужно платить / какие есть тарифы?
    Сейчас сервис находится на стадии бета-теста и воспользоваться им можно совершенно бесплатно. Бета-тест продлится до конца сентября.
  • Как пополняется база разработчиков?
    База разработчиков ежедневно пополняется благодаря: 1) каждому поиску, совершённому в GitJob. Например, вы начали искать Python-разработчика в Москве. Система сканирует профили Github и Stack Overflow и подбирает подходящие. Вы получаете список кандидатов, который в это же время пополняет нашу базу разработчиков Python из Москвы. И так каждый новый поиск, совершённый на сайте, пополняет существующие базы; 2) также база будет пополняться по мере того, как разработчики сами будут регистрировать свои профили на GitJob, чтобы получать более точные и релевантные офферы.
  • Как вы можете гарантировать сохранность данных, если утверждаете, что сканируете репозитории? Это безопасно?
    GitJob анализирует только манифест-файл, мы не парсим и не сохраняем ваш код. Сервис доступен для проектов с закрытыми репозиториями.
  • Есть ручной поиск разработчиков по языкам / библиотекам?
    Да, есть и такой режим поиска, где вы указываете необходимые языки / библиотеки. Далее система подбирает вам кандидатов, исходя из анализа репозиториев в их профилях на Github и Stack Overflow.
